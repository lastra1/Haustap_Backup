# Laravel Deployment Guide for HausTap

## Overview
This is your **Laravel PHP project** with Filament admin panel that needs to be deployed to Render.

## Project Structure
- **Framework**: Laravel 12.x with PHP 8.2
- **Admin Panel**: Filament 3.x
- **Database**: MySQL
- **Authentication**: Laravel Sanctum
- **Location**: `Haustap_Capstone-Haustap_Connecting/Haustap_Capstone-Haustap_Connecting/backend/`

## Environment Variables Required

### Database (Provided by Render)
```bash
DB_CONNECTION=mysql
DB_HOST=[Render provides]
DB_PORT=3306
DB_DATABASE=[Render provides]
DB_USERNAME=[Render provides]
DB_PASSWORD=[Render provides]
```

### Application Settings
```bash
APP_NAME="HausTap Service Booking Platform"
APP_ENV=production
APP_DEBUG=false
APP_URL=[Render provides]
APP_KEY=[Auto-generated by Render]
```

### Laravel Configuration
```bash
SESSION_DRIVER=database
SESSION_LIFETIME=120
SESSION_ENCRYPT=true
CACHE_STORE=database
QUEUE_CONNECTION=database
FILESYSTEM_DISK=local
LOG_CHANNEL=stderr
```

### Mail Settings (Mailtrap)
```bash
MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=[Add manually]
MAIL_PASSWORD=[Add manually]
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=noreply@haustap.com
MAIL_FROM_NAME="${APP_NAME}"
```

## Deployment Steps

1. **Add Mailtrap Credentials** (Optional)
   - Sign up at https://mailtrap.io
   - Add MAIL_USERNAME and MAIL_PASSWORD to Render

2. **Deploy**
   - Push to GitHub → Render automatically deploys
   - Monitor build logs for any issues

3. **Post-Deployment**
   - Access Filament admin at: `https://your-service.onrender.com/admin`
   - Test API endpoints at: `https://your-service.onrender.com/api/health`

## Admin Access
After deployment, you'll need to create an admin user:
```bash
php artisan tinker
>>> $user = new App\Models\User();
>>> $user->name = 'Admin';
>>> $user->email = 'admin@haustap.com';
>>> $user->password = bcrypt('your-password');
>>> $user->save();
```
